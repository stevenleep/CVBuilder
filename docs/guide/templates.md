# 模板管理

## 模板类型

CVBuilder 支持两种类型的模板：

### 1. 组件模板（Material Templates）

单个物料或物料组合的模板。

**用途**:
- 保存常用的组件配置
- 复用自定义的物料组合
- 快速添加标准化内容

**示例**:
- 工作经历条目模板
- 教育背景模板
- 个人信息模板

### 2. 简历模板（Resume Templates）

完整的简历模板。

**用途**:
- 快速创建新简历
- 切换不同风格的简历
- 为不同岗位准备不同版本

**示例**:
- 互联网行业简历模板
- 传统企业简历模板
- 应届生简历模板
- 资深工程师简历模板

## 使用组件模板

### 保存组件模板

#### 方法 1: 右键菜单

1. 选中要保存的物料
2. 右键 → **保存为模板**
3. 输入模板信息：
   - 模板名称（必填）
   - 描述（可选）
   - 标签（可选）
4. 点击确认

#### 方法 2: 属性面板

1. 选中物料
2. 在属性面板底部找到"保存为模板"
3. 输入模板信息
4. 保存

### 使用组件模板

#### 从物料面板

1. 切换到**自定义模板**标签
2. 找到要使用的模板
3. 拖拽到画布或双击添加

#### 从模板菜单

1. 点击工具栏的"模板"按钮
2. 选择"组件模板"
3. 浏览并应用模板

### 管理组件模板

#### 编辑模板

1. 在自定义模板列表中找到模板
2. 点击编辑按钮
3. 修改名称、描述或内容
4. 保存更改

#### 删除模板

1. 找到要删除的模板
2. 点击删除按钮
3. 确认删除

**注意**: 删除操作不可恢复，请谨慎操作。

#### 复制模板

1. 找到要复制的模板
2. 点击复制按钮
3. 会创建一个新的副本
4. 可以在副本基础上修改

## 使用简历模板

### 创建简历模板

#### 从当前简历保存

1. 完成简历编辑
2. 点击工具栏的"模板"按钮
3. 选择"保存为简历模板"
4. 输入模板信息：
   - 模板名称
   - 模板描述
   - 适用场景
   - 标签
5. 确认保存

#### 模板包含的内容

- 完整的页面结构
- 所有物料配置
- 当前主题设置
- 布局和样式

### 应用简历模板

#### 新建简历

1. 打开编辑器
2. 点击"简历模板"面板
3. 浏览可用模板
4. 点击"使用此模板"
5. 模板内容加载到画布

#### 替换当前简历

1. 点击"简历模板"面板
2. 选择模板
3. 点击"应用"
4. 确认替换（会覆盖当前内容）

**警告**: 应用模板会替换所有当前内容，建议先保存当前工作。

### 管理简历模板

#### 预览模板

1. 在模板列表中找到模板
2. 鼠标悬停查看缩略图
3. 点击"预览"查看详细内容

#### 编辑模板信息

1. 找到模板
2. 点击"编辑"
3. 修改名称、描述、标签
4. 保存更改

**注意**: 只能编辑模板信息，不能编辑模板内容。要更新模板内容，需要重新保存。

#### 删除模板

1. 找到要删除的模板
2. 点击删除按钮
3. 确认删除

#### 导出模板

1. 选择模板
2. 点击"导出"
3. 下载 JSON 文件
4. 可以分享给他人或备份

#### 导入模板

1. 点击"导入模板"
2. 选择 JSON 文件
3. 模板添加到模板库

## 模板最佳实践

### 1. 命名规范

使用清晰、描述性的名称：

```
✅ 好的命名:
- "互联网公司 - 前端工程师"
- "工作经历 - 项目型"
- "教育背景 - 硕士研究生"

❌ 不好的命名:
- "模板1"
- "我的简历"
- "template"
```

### 2. 添加详细描述

```
✅ 好的描述:
适用于互联网公司的前端工程师职位，突出项目经验和技术栈。
包含5年工作经验、3个重点项目、技能评级展示。

❌ 不好的描述:
简历模板
```

### 3. 使用标签分类

```
标签示例:
- 行业: 互联网、金融、教育、医疗
- 职位: 工程师、设计师、产品、运营
- 经验: 应届生、1-3年、3-5年、5年以上
- 风格: 简约、商务、创意、学术
```

### 4. 定期更新

- 及时更新过时的信息
- 根据反馈改进模板
- 删除不再使用的模板

### 5. 版本管理

```
建议方案:
1. 为每个岗位类型创建基础模板
2. 根据具体公司微调后另存为新版本
3. 使用标签区分: "v1", "v2", "2024版"
```

## 模板示例

### 组件模板示例

#### 工作经历模板 - 项目型

```json
{
  "name": "工作经历 - 项目型",
  "description": "强调项目经验的工作经历模板",
  "schema": {
    "type": "ExperienceItem",
    "props": {
      "company": "XX科技有限公司",
      "position": "前端工程师",
      "startDate": "2020-01",
      "endDate": "2023-12",
      "description": "负责公司核心产品的前端开发工作",
      "achievements": "• 主导XX项目重构，性能提升50%\n• 搭建前端基础设施，提高团队效率\n• 指导3名初级工程师成长"
    }
  }
}
```

#### 教育背景模板 - 研究生

```json
{
  "name": "教育背景 - 硕士研究生",
  "description": "研究生教育背景模板，包含科研成果",
  "schema": {
    "type": "EducationItem",
    "props": {
      "school": "XX大学",
      "major": "计算机科学与技术",
      "degree": "硕士",
      "startDate": "2018-09",
      "endDate": "2021-06",
      "gpa": "3.8/4.0",
      "scholarships": "一等奖学金、国家奖学金",
      "achievements": "发表SCI论文2篇，参与国家级项目1项"
    }
  }
}
```

### 简历模板示例

#### 互联网公司 - 前端工程师

**适用对象**: 3-5年经验的前端工程师  
**特点**: 
- 突出项目经验和技术栈
- 简约现代的设计风格
- 技能评级可视化

**包含模块**:
- 个人信息（带头像）
- 工作经历（3-4段）
- 项目经验（2-3个重点项目）
- 技能列表（评级展示）
- 教育背景
- 个人优势

#### 应届生简历模板

**适用对象**: 应届毕业生  
**特点**:
- 突出教育背景和实习经历
- 强调学习能力和潜力
- 清晰的布局

**包含模块**:
- 个人信息
- 教育背景（详细）
- 实习经历
- 项目经验（学校项目）
- 技能证书
- 获奖经历
- 自我评价

## 模板数据结构

### 组件模板

```typescript
interface MaterialTemplate {
  id: string
  name: string
  description?: string
  schema: NodeSchema
  thumbnail?: string
  createdAt: number
  updatedAt: number
  tags?: string[]
}
```

### 简历模板

```typescript
interface ResumeTemplate {
  id: string
  name: string
  description?: string
  pageSchema: PageSchema
  themeConfig?: ITheme
  thumbnail?: string
  createdAt: number
  updatedAt: number
  tags?: string[]
  author?: string
  category?: string
}
```

## 模板存储

模板保存在浏览器的 IndexedDB 中：

- **组件模板**: `templates` 存储空间
- **简历模板**: `resume-templates` 存储空间

### 备份模板

建议定期导出模板进行备份：

```bash
1. 点击"导出所有模板"
2. 保存 JSON 文件
3. 存储在云盘或本地
```

### 恢复模板

```bash
1. 点击"导入模板"
2. 选择备份的 JSON 文件
3. 确认导入
```

## 常见问题

### Q: 模板数量有限制吗？

A: 理论上没有限制，但建议保持在50个以内，以便管理。

### Q: 可以分享模板吗？

A: 可以。导出模板为 JSON 文件，分享给他人导入即可。

### Q: 模板会丢失吗？

A: 模板保存在浏览器本地，清除浏览器数据会丢失。建议定期备份。

### Q: 如何批量删除模板？

A: 当前版本不支持批量删除，需要逐个删除。

### Q: 模板可以跨浏览器使用吗？

A: 不能直接跨浏览器。需要导出后在其他浏览器导入。
